# Module-Dokumentation

Pro Modul (1–18, siehe [SPEC.md](../../SPEC.md) §4) liegt hier eine `*.md`-Datei
mit:

- **Was das Modul tut** (1 Absatz)
- **Datenmodell** (Prisma-Entities, die dazugehören)
- **API-Endpoints** (Referenz auf [specs/api.md](../../specs/api.md))
- **UI-Seiten** (Referenz auf Route + Komponenten in `apps/web`)
- **Feature-Flags** (welche Flags gibt es dafür)
- **Integrationen** (welche externen Provider werden aufgerufen)
- **Testing-Notes** (was gedeckt ist, was offen)
- **Open Questions** (was noch unklar ist)

Diese Dateien werden **während** der Arbeit am Modul geschrieben, nicht
nachträglich. Jedes Modul bekommt in Phase 1 seine eigene Datei, wenn es
angefasst wird.
